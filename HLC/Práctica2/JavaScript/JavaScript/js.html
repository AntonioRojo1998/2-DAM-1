JavaScript: Qué es y para qué se utiliza

JavaScript es un lenguaje de programación para implementar funcionalidades complejas en una página web. Una página web de hoy en día está compuesta por tres capas: HTML, CSS y JavaScript.
Estas tres tecnologías son las fundamentales para cualquier página que se aprecie hoy en día.
El código HTML contiene la estructura y el texto; el CSS adorna y colorea el HTML; y JavaScript añade la funcionalidad y el movimiento al HTML. Ese es el objetivo de JavaScript en entornos web.

JavaScript se vuelve mucho más potente si usamos APIs (Application Programming Interfaces). Una API son bloques de código listos para implementar en tu script. Puedes imaginarlas como si fueran piezas de un ordenador: imaginemos que quieres montar un ordenador. No vas a comprar los chips, transistores, condensadores y demás elementos electrónicos, y soldarlos a una placa tú mismo, ¿verdad? Lo suyo sería comprar una placa base, tarjeta gráfica, CPU, etc. y montar las piezas en los sitios que deben ocupar. Seamos realistas, si tuviéramos que construir nuestros ordenadores de la primera forma, no creo que muchos tuviéramos uno. Pues lo mismo pasa con las APIs, son las piezas ya construidas, siendo posible montarlas donde queramos.

Las APIs tienen dos categorías: APIs de terceros o APIs del navegador. Las APIs del navegador están ya incluidas en el propio navegador (duh), lo cual las hace de fácil acceso, además de permitir acceder a datos del entorno del ordenador.
Algunos ejemplos de APIs del navegador son:

  • La API DOM (Document Object Model) te permite manipular el HTML y el CSS de una página, aplicando un estilo dinámico.
  • La API Geolocation se usa para obtener información de, como su propio nombre indica, geolocalización.
  • Las APIs de Canvas y WebGL se usan para crear animaciones y gráficos en 2D y 3D.
  • APIs de vídeo y sonido, como HTMLMediaElement y WebRTC, te permiten animar tu web con sonidos y vídeo.
Un ejemplo básico

A continuación veremos un pequeño ejemplo de lo que puede hacer JavaScript. El ejemplo consiste en lo que tú quieras.

Usando el siguiente campo de texto, botones y lista, puedes modificar una frase del párrafo anterior. Adelante, pruébalo.


Echémosle un vistazo al código responsable de estos cambios, paso a paso.
Primero veamos las variables: 

// Variables para los elementos del documento
const texto = document.querySelector(".campoModificar");
const botonModificar = document.querySelector(".botonModificar");
const listaModificar = document.querySelector(".listaModificar");
const textoModificable = document.querySelector(".textoModificable");

// Para mostrar y ocultar texto
const titulos = document.querySelectorAll("article-title");
const contenido = document.querySelectorAll("article-content");

Como podemos ver, no es necesario indicar el tipo de elemento que van a almacenar estas variables. Simplemente indicamos si van a ser constantes (con const) o no constantes (con let o var).
Las variables que se usan aquí están almacenando objetos DOM (anteriormente mencionados), y son básicamente objetos que representan etiquetas HTML. En el caso de las variables titulos y contenido son arrays de objetos DOM.

Ahora echémosle un vistazo a las funciones utilizadas: 

function cambiarTexto() {
    textoModificable.textContent = texto.value;
}

function cambiarColor(index) {
    let bgcol = "white";
    let col = "black"
    switch (index) {
        case 0:
            bgcol = "black";
            col = "white";
            break;
        case 1:
            bgcol = "#daa0a0";
            col = "black";
            break;
        case 2:
            bgcol = "#bedaa0";
            col = "black";
            break;
        case 3:
            bgcol = "#a0a0da";
            col = "black";
            break;
    }
    listaModificar.style.color = col;
    textoModificable.style.color = col;

    listaModificar.style.backgroundColor = bgcol;
    textoModificable.style.backgroundColor = bgcol;
}

function toggleInfo(id) {
    let state = contenido[id].style.display;
    if (state === 'none') {
        contenido[id].style.display = "block";
        titulos[id].textContent =  titulos[id].textContent.replace("(clic para mostrar)", "");
    } else {
        titulos[id].textContent += " (clic para mostrar)";
        contenido[id].style.display = "none";
    }
}

Aquí tenemos 3 funciones:

  • Una función que cambia el texto de uno de los objetos que tenemos almacenados en una variable.
  • Otra que cambia el color de la lista desplegable y del texto que se cambia.
  • Y otra que esconde y muestra el contenido de los artículos.

Vamos a estudiarlas una a una:

Función cambiarTexto()

Esta función cambia el contenido textual de la etiqueta clasificada como "textoModificable" por el contenido textual almacenado en la etiqueta clasificada como "campoModificar". Es decir, recupera el texto escrito en el elemento input y lo pone en la etiqueta clasificada como "textoModificado".

Función cambiarColor(index)

Con el parámetro index le indicamos a la función qué elemento de la lista desplegable se ha escogido, y dependiendo de cuál sea se cambiarán los atributos color y backgroundColor de la lista y del texto modificable.

Función toggleInfo(id)

Esta función usa los dos arrays de DOMs, titulos y contenido. El contenido de esta página está estructurado por artículos, los cuales tienen un título y un contenido. Estos dos arrays almacenan los cuatro títulos y secciones de contenido que hay en esta página.
Con el parámetro id se le indica a la función qué título se ha hecho clic en, y esconderá/mostrará el contenido respectivo de dicho título. El cambio de esconder o mostrar lo hará modificando el valor del atributo display.
Además, esta función modificará el contenido textual del título, añadiendo un (clic para mostrar) si el contenido se encuentra oculto.

Y por último, tenemos que decirle al código que preste atención al usuario en ciertas partes. Esto lo haremos con event listeners:

botonModificar.addEventListener("click", cambiarTexto);
listaModificar.addEventListener("click", function () {
    cambiarColor(listaModificar.selectedIndex)
});

titulos[0].addEventListener("click", function () {
    toggleInfo(0)
});
titulos[1].addEventListener("click", function () {
    toggleInfo(1)
});
titulos[2].addEventListener("click", function () {
    toggleInfo(2)
});
titulos[3].addEventListener("click", function () {
    toggleInfo(3)
});

La función addEventListener toma un tipo de evento ("click") y una función, ya sea un nombre de función (sin dar los parámetros), o una función anónima (que puede llamar a una función con parámetros).

Cómo añadir JS a tu página

Existen varias formas de implementar JS a tu página. Se puede hacer de forma interna o externa, es decir, en el propio archivo HTML o en un archivo externo de tipo .js.

JavaScript interno

Para añadir un bloque de código JS en un archivo HTML, se usa la etiqueta <script>. Entre estas etiquetas se puede escribir código JavaScript, que se ejecutará cuando se llegue a la sección del HTML donde se encuentre el código. Esto, por desgracia, puede provocar errores si el código está intentando acceder a contenido del HTML que aún no se ha cargado. Más adelante hablaremos de las posibles soluciones a este problema, por el momento simplemente estudiaremos el código.

Estrategias para cargar el script

Existen varios problemas a la hora de cargar los scripts, sobretodo si queremos que se ejecuten en ciertos momentos. Un problema común es el orden en el que carga el código HTML. Como ya deberías saber, el código HTML se carga en el orden en el que está escrito, por lo que si una etiqueta <script> se encuentra entre las etiquetas <head>, se ejecutará antes que todo lo que haya en el cuerpo del HTML. Y si nuestro código JavaScript quiere acceder a código HTML que aún no está cargado, nos dará problemas.
Para evitar este problema, podemos simplemente añadir el código JS al final de la etiqueta <body>. Esta estrategia, sin embargo, viene acompañada de sus propios problemas. El más importante, y el que más impacto tiene sobre la experiencia del usuario, es que no se cargará el código JS hasta que se haya cargado todo el HTML, lo que significa que se cargará de golpe. En páginas con grandes cantidades de código JS, esto puede congelar la página hasta que termine de cargar el JS, empeorando la experiencia de usuario.
Una solución para no colocar la etiqueta <script> al final del cuerpo del HTML es aplicar un event listener al documento, para que una vez termine de cargar todo el HTML se ejecute el JS. Así se puede añadir la etiqueta <script> donde queramos.
La solución anterior es aplicada a código JS interno. En caso de que estemos usando código externo, existe una solución mejor: usar el atributo defer en la etiqueta <script>. Al añadir este atributo, el código JS se carga a la vez que el HTML, y se ejecuta una vez se haya terminado de parsear el HTML.

En ocasiones, un archivo HTML usará varios scripts. En estos casos tendremos que decidir si queremos que se carguen los scripts a la vez, o en orden. Si queremos que se carguen en orden, usaremos el atributo defer en todos los scripts. En el caso de que no importe el orden de ejecución (por ejemplo en el caso en el que los scripts no interfieran los unos con los otros), usaremos el atributo async.
