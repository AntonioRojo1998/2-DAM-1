Git is a tool that performs a function of code version control in a distributed way, of which we highlight several features:

  • It is free software.
  • It is very powerful.
  • It is very fast.
  • It was designed by Linus Torvalds.
  • It does not depend on a central repository.
  • With it we can keep a complete version history.
  • We can move, as if we had a pointer in time, through all the code reviews and scroll in a very agile way.
  • It has a branch work system that makes it especially powerful.
  • Regarding the functionality of the branches, they are intended to provoke divergent projects from a main project, to carry out experiments or to test new functionalities.
  • The branches can have a different progress line from the main branch where the core of our development is. At some point we can get to test some of those improvements or changes in the code and make a merge to our main project, since all this is handled by Git in a very efficient way.

What is a distributed version control system?

There is no server that maintains a copy of the repository, but it is maintained between the clients that are in use of the repository and. For example, like P2P clients, the more developers have connected, the better connectivity there will be between all.
